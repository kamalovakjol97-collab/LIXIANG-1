# Полная структура сайта XGL и рекомендации по улучшению

## Технологический стек
- **React** (Vite + JSX)
- **React Router** (маршрутизация)
- **Context API** (LanguageContext — RU/ZH)
- **Supabase** (форма заявки)
- **CSS** (переменные в `index.css`, компонентные стили)

---

## Общая структура приложения

```
App
├── LanguageProvider (контекст языка)
└── Router
    └── Routes
        ├── Каждая страница: TopBar → Header → [Контент] → Footer
        └── Маршруты: /, /services, /news, /projects, /history, /contacts, /faq, /privacy, /lk
```

**Общие компоненты на всех страницах:**
- **TopBar** — верхняя панель (курсы валют, язык, контакты)
- **Header** — логотип, навигация, кнопка «Личный кабинет XGL» → `/lk`
- **Footer** — компания, email, часы работы, ссылка на политику конфиденциальности, © 2024

## Места для улучшения

### Критичные / логика

1. **Страница услуг: кнопка «Рассчитать стоимость» и карточки**
   - Сейчас: `scrollToForm()` ищет `#application-form` на странице; формы там нет → скролл ничего не делает.
   - **Варианты:** либо добавить блок с формой заявки в конец страницы услуг, либо при клике перенаправлять на главную с якорем `/#application-form`, либо явно вести на отдельную страницу «Оставить заявку».

2. **Footer: ссылка «Политика конфиденциальности»**
   - Используется `<a href="/privacy">` — при переходе полная перезагрузка.
   - **Рекомендация:** заменить на `<Link to="/privacy">` из React Router для SPA-навигации.

3. **Footer: перевод «privacy»**
   - В коде используется `t('privacy')`, в `LanguageContext` ключа `privacy` нет — отображается ключ «privacy».
   - **Рекомендация:** добавить в `translations` для `ru` и `zh` ключ `privacy` («Политика конфиденциальности» / «隐私政策»).

4. **Политика конфиденциальности: только русский**
   - Контент только на RU; переключатель языка на сайте есть.
   - **Рекомендация:** добавить китайскую версию текста и показывать её через `language` из контекста.

5. **Личный кабинет**
   - Сейчас заглушка. Для продакшена нужны: авторизация, список заявок, отслеживание грузов и т.д.

---

### UX и навигация

6. **Мобильное меню (Header)**
   - На ширине ≤1200px навигация скрыта (`display: none`), гамбургер не реализован.
   - **Рекомендация:** добавить бургер-меню, выезжающую панель или выпадающее меню с теми же пунктами и кнопкой «Личный кабинет».

7. **Сохранение языка**
   - Язык хранится только в state — при перезагрузке сбрасывается на RU.
   - **Рекомендация:** сохранять выбор в `localStorage` (или cookie) и инициализировать язык из него.

8. **Страница 404**
   - Нет обработки несуществующих путей.
   - **Рекомендация:** добавить `<Route path="*" element={<NotFoundPage />}>` с ссылкой на главную и поиском.

---

### Контент и доступность

9. **Год в Footer**
   - Зашито «© 2024». Со временем устареет.
   - **Рекомендация:** выводить текущий год: `© ${new Date().getFullYear()}`.

10. **Доступность (a11y)**
    - Не везде есть `aria-label`, `role`, корректные заголовки (h1 → h2 → h3).
    - **Рекомендация:** один h1 на страницу, логичная иерархия заголовков, подписи к интерактивным элементам, фокус при навигации с клавиатуры.

11. **SEO**
    - Нет отдельных мета-тегов (title, description) для страниц.
    - **Рекомендация:** использовать React Helmet (или аналог) и задавать title/description для каждой страницы.

---

### Код и архитектура

12. **Дублирование разметки маршрутов**
    - TopBar, Header, Footer повторяются в каждом `Route`.
    - **Рекомендация:** вынести общий layout в компонент (например, `MainLayout`) и рендерить в нём `<Outlet />` для контента страницы.

13. **Переводы**
    - Много строк захардкожены в компонентах (RU/ZH через тернарники).
    - **Рекомендация:** собрать все строки в один объект переводов (или файлы ru.json / zh.json) и использовать `t('key')` везде — проще поддерживать и добавлять языки.

14. **Компонент Services**
    - На странице услуг используется только разметка в формате карусели внутри `ServicesPage.jsx`; компонент `Services.jsx` (двухколоночный интерфейс с табами) не используется.
    - **Рекомендация:** либо удалить неиспользуемый `Services.jsx` (и `Services.css`), либо явно решить, где он должен использоваться, чтобы не путать структуру.

15. **Обработка ошибок формы заявки**
    - При ошибке Supabase показывается общее сообщение; детали не логируются для пользователя.
    - **Рекомендация:** показывать пользователю короткое сообщение об ошибке и при необходимости кнопку «Повторить»; полные ошибки — в консоль или систему мониторинга.

---

### Визуал и производительность

16. **Внешние изображения**
    - Используются URL Unsplash и Wikipedia; при их недоступности блоки могут пустовать или долго грузиться.
    - **Рекомендация:** по возможности хранить критичные изображения в проекте (или CDN), использовать fallback/placeholder и при необходимости lazy loading.

17. **Hero на внутренних страницах**
    - На части страниц hero с однотонным фоном; на услугах уже есть фон + градиент.
    - **Рекомендация:** для единообразия задать единый подход (например, везде фон + лёгкий overlay или везде однотон) и вынести в общий компонент `PageHero` с пропсами (title, breadcrumbs, background).
